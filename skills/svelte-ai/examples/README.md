# Examples

Working snippets that show how to assemble the Svelte AI toolkit in real projects.

| File | Use it when |
|---|---|
| `agents-md.md` | Writing or shipping an `AGENTS.md` (or `CLAUDE.md` / `GEMINI.md`) contract for the agent |
| `local-setup.md` | Wiring up the local stdio MCP server for Claude Code / Desktop, Codex, Copilot, Antigravity, OpenCode, VS Code, Cursor, Zed |
| `remote-setup.md` | Pointing an AI client at `https://mcp.svelte.dev/mcp` |
| `mcp-tools.md` | Calling `list-sections` / `get-documentation` / `svelte-autofixer` / `playground-link` from a session |
| `mcp-prompts.md` | Submitting the `svelte-task` prompt as a user message |
| `cli-usage.md` | Driving `@sveltejs/mcp` from the shell, in scripts, or in pre-commit hooks |
| `subagents.md` | Invoking `svelte-file-editor` (and the `svelte-code-writer` / `svelte-core-bestpractices` skills) |

Read top-to-bottom the first time; after that, jump directly to the file matching your current task.
