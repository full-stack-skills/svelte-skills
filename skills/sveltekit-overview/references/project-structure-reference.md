# Project Structure Reference

Every file and folder in a standard SvelteKit project, what it's for, and when you need it.

## Top-level layout

```tree
my-project/
├ src/
├ static/
├ tests/
├ package.json
├ svelte.config.js
├ tsconfig.json
└ vite.config.js
```

Plus the auto-generated `.svelte-kit/` (gitignored) and the `.svelte-kit/tsconfig.json` it produces.

## src/

The meat of your project. Everything except `src/routes` and `src/app.html` is optional.

### src/lib/

Your library code. Importable via the `$lib` alias.

- `src/lib/server/` — server-only code. Import via `$lib/server`. SvelteKit blocks client imports.
- `src/lib/components/`, `src/lib/utils/`, etc. — your own subfolders.

### src/params/

Param matchers. A matcher is a function `match(param: string): boolean` exported from `src/params/<name>.ts`. Used as `[slug=<name>]` in route folder names.

### src/routes/

The filesystem router. Each folder is a URL segment. See `routing-reference.md`.

### src/app.html

The HTML template. Placeholders:

| Placeholder | Purpose |
|---|---|
| `%sveltekit.head%` | `<link>`/`<script>` tags + `<svelte:head>` content |
| `%sveltekit.body%` | Rendered page markup. MUST be inside a `<div>` or other element |
| `%sveltekit.assets%` | Path to static assets (respects `paths.assets` and `paths.base`) |
| `%sveltekit.nonce%` | CSP nonce (only if `csp` is enabled) |
| `%sveltekit.env.[NAME]%` | Public env vars (must start with `PUBLIC_` or be declared public) |
| `%sveltekit.version%` | App version from config |

### src/error.html

Static fallback rendered when `+error.svelte` is unavailable. Placeholders: `%sveltekit.status%`, `%sveltekit.error.message%`.

### src/hooks.client.js / hooks.server.js

Client and server hooks. Use `Handle`, `HandleServerError`, `HandleClientError` types.

### src/service-worker.js

Service worker source. Registered automatically when `serviceWorker.register: true`. Bump the version to bust caches.

### src/instrumentation.server.js

Observability setup (requires adapter support). Runs before your app code.

## static/

Files served verbatim (no transformation, no hashing). Place `robots.txt`, `favicon.ico`, etc. here. Prefer `import` for images so Vite can hash them.

## tests/

Browser tests live here if you added Playwright during `npx sv create`. Each `*.spec.ts` file is a test file. `test.ts` is global setup.

## package.json

- `type: "module"` — `.js` files are ESM. CommonJS files need `.cjs`.
- `devDependencies` should include `@sveltejs/kit`, `svelte`, `vite`.
- Scripts: `dev` (Vite dev server), `build` (Vite build), `preview` (preview the built output).

## svelte.config.js

Exports a `SvelteConfig` object. See `examples/svelte-config.md`. At minimum: `kit.adapter`.

## tsconfig.json

Should extend `.svelte-kit/tsconfig.json` (auto-generated by SvelteKit). To customize `include`/`exclude`, override via `typescript.config` in `svelte.config.js`.

```json
// tsconfig.json
{
  "extends": "./.svelte-kit/tsconfig.json",
  "compilerOptions": {
    "allowJs": true,
    "checkJs": true,
    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "skipLibCheck": true,
    "sourceMap": true,
    "strict": true,
    "moduleResolution": "bundler"
  }
}
```

## vite.config.js

```js
import { sveltekit } from '@sveltejs/kit/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [sveltekit()]
});
```

A SvelteKit project is a Vite project. Anything Vite supports works (CSS preprocessors, MDX, image plugins, etc.).

## .svelte-kit/

Generated. Holds `tsconfig.json`, type definitions for `$types`, the dev server output, and build artifacts. Safe to delete; it regenerates.

## Other common files (not part of SvelteKit)

- `.gitignore` — must include `.svelte-kit`, `build`, `node_modules`.
- `.npmrc` — package manager config.
- `eslint.config.js` / `.prettierrc` — added by `sv add`.
- `Dockerfile` — if containerizing.

## Routes folder conventions

```tree
src/routes/
├ +layout.svelte         # root layout
├ +layout.js             # root load
├ +page.svelte           # /
├ +error.svelte          # 404 + root error boundary
├ about/
│  └ +page.svelte        # /about
└ blog/
   ├ +page.svelte        # /blog
   └ [slug]/
      ├ +page.svelte
      ├ +page.server.js
      └ +error.svelte    # error boundary for /blog/[slug]
```

Files NOT starting with `+` are ignored by the router — use them to colocate route-specific components or utilities.
